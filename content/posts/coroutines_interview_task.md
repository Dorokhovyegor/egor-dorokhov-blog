Привет!

Хочу разобрать задачу с собеседования. Она целиком и полностью про корутины, поэтому если ты видишь их впервые, лучше сначала почитать официальную документацию [здесь](https://kotlinlang.org/docs/coroutines-overview.html).

*- Через курсив буду задавать потенциальные вопросы как от собеседующего.*

Задача выглядит так:

Что напечатает данный кусок кода?

    fun main() = runBlocking { 
        val a = launch { 
            while (true) { 
                Thread.sleep(1000) 
                println("Show Loading") 
            } 
        } 
        
        launch { 
            withContext(Dispatchers.IO) { 
                delay(2500) 
                a.cancel() 
                println("Finished") 
            } 
        } 
    }

Прежде чем углубляться в код, попробуй посмотреть на него «наискосок». Здесь есть функция main(), которая вызывает билдер [runBlocking()](https://kotlinlang.org/docs/coroutines-basics.html#runblocking).

Билдер запускает корутину, которая блокирет поток, до тех пор, пока не завершится.

*- На каком потоке будет исполняться runBlocking?*

Это поток main. Самый обычный поток. Ты можешь в этом убедиться, если выведешь его имя через print(Thread.currentThread().name).

Внутри корутины запускаются две дочерние через билдер launch().

Первая дочерня корутина (пусть будет А) запускает бесконечный цикл: через Thread.sleep(1000) она «усыпляет» поток на секунду и затем выводит сообщение “Show Loading”.

Вторая дочерняя корутина (пусть будет Б) переключает контекст на Dispatcher.IO, то есть начинает выполняться в пуле потоков этого диспетчера. Затем делает задержку через delay на 2.5 секунды, отменяет первую корутину и выводит сообщение “Finished”.

*- Что здесь произойдет, если мы запустим код?*

Кажется, ответ очевиден: два раза напечатается “Show Loading”, затем первая корутина отменится через a.cancel(), и ты увидишь “Finished”.

**(это не так)**. Давай рассуждать.

Если ты запустишь этот код, то увидишь, что “Show Loading” печатается гораздо больше двух раз — фактически бесконечно.

Проблема в том, что корутина "А" после запуска никогда не освобождает поток main. Цикл while(true) бесконечный, и после каждого Thread.sleep(1000) поток просыпается только для того, чтобы напечатать “Show Loading”, и снова засыпает. В результате,  корутина "Б" даже не получает шанса запуститься.

Фиксируем, корутина "А" выполняется на потоке main и полностью его блокирует навсегда.

*- Хорошо, а что если запустить корутину "А" с Dispatcher.IO?*

В этом случае ситуация изменится: ты увидишь “Show Loading” два раза, затем “Finished”. Но после этого “Show Loading” снова начнет печататься бесконечно. 

*- Почему так происходит?*

Следующая проблема — корутина "А" никак не реагирует на отмену. Корутины нельзя просто «убить» извне — cooperative cancellation означает, что им можно только сообщить об отмене. А уже сама корутина должна корректно завершиться.

*- Как сообщить корутине об отмене?*

Есть несколько способов, с помощью которых корутина может узнать об отмене:

1. Suspension point (все suspend-функции из kotlinx.coroutines — внутри них есть проверки отмены)
2. Явные проверки (isActive, ensureActive(), yield())

Исходя из этого, внутри цикла стоит заменить while(true) на while(isActive). Тогда перед каждой новой итерацией цикла корутина "А" будет проверять, не отменила ли ее "Б".

После этого изменения код будет выглядеть так:

    fun main() = runBlocking { 
        val a = launch(Dispatchers.IO) { 
            while (isActive) { 
                Thread.sleep(1000) 
                println("Show Loading") 
            } 
        } 
        
        launch { 
            withContext(Dispatchers.IO) { 
                delay(2500) 
                a.cancel() 
                println("Finished") 
            } 
        } 
    } 


*- Запусти этот вариант, и ты увидишь: “Show Loading” выводится два раза, затем “Finished”, а потом — ещё один “Show Loading”. Откуда он берется?*

Давай буквально «прогоним» выполнение корутины "A":

* Запускаюсь
* Меня отменили? — нет
* Ухожу спать на 1 секунду (прошла 1 секунда)
* Печатаю Show Loading
* Меня отменили? — нет
* Ухожу спать на 1 секунду (прошло 2 секунды)
* Печатаю Show Loading
* Меня отменили? — нет
* Ухожу спать на 1 секунду (прошло 3 секунды) — где-то здесь меня отменяют
* Просыпаюсь и печатаю Show Loading
* Меня отменили? — да
* Завершаюсь

Теперь видно, откуда берется лишний вывод.

Самое простое решение — заменить Thread.sleep(1000) на delay(1000). Функция delay() умеет проверять отмену и не блокирует поток, в отличие от Thread.sleep(). Благодаря этому вторая корутина сможет выполниться.

Из этого следует, что Dispatcher.IO для первой корутины больше не нужен. Проверку isActive тоже можно убрать, хотя она делает условие цикла более явным.

Итоговый вариант будет выглядеть так:

    fun main(): Unit = runBlocking {  
        val a = launch {  
            while (true) {  
                delay(1000)  
                println("Show Loading")  
            }  
        }  
        
        launch {  
            withContext(Dispatchers.IO) {  
                delay(2500)  
                a.cancel()  
                println("Finished")  
            }  
        }
    }
